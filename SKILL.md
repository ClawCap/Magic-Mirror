## 核心概念

### 什么是"照妖镜报告"？

不是用户画像。不是人格测试。

**是从用户社交数据中挖掘「反差」「矛盾」「隐藏真相」，用毒舌但不冒犯的方式呈现出来。**

好的照妖镜报告让人看了反应是：
- "靠，被看穿了"
- "哈哈哈哈哈我怎么会这样"
- "这个太准了我要发朋友圈"

坏的照妖镜报告让人看了反应是：
- "嗯……挺准的"（太正经，没意思）
- "这分析跟我有什么关系"（太泛，没有个人化）
- "感觉被冒犯了"（过线了）

### 分析维度

照妖镜聚焦在**最有戏剧性的反差点**：

| 维度 | 找什么 | 怎么找 |
|------|-------|-------|
| 🎭 **人设 vs 真相** | 公开展示 vs 收藏/点赞暴露的真实偏好 | 对比发帖 vs 收藏/点赞 |
| 📊 **嘴上 vs 行动** | 说要做的 vs 实际做到的 | 豆瓣"想读"vs"已读"等 |
| 🔍 **意识 vs 潜意识** | 自认为的兴趣 vs 真实兴趣 | 关注列表聚类、内容分布 |
| 👥 **跨平台人设** | 不同平台的不同面孔 | 跨平台风格对比 |
| 📈 **自以为 vs 真正** | "我是XX的人" vs 数据反驳 | 自我描述 vs 实际行为 |

在以下位置查找 ManoBrowser 的 SKILL.md 文件（按优先级）：

2. `~/.openclaw/skills/manobrowser/SKILL.md`

- **未找到** → 自动从 GitHub 下载：
  ```bash
  git clone https://github.com/ClawCap/ManoBrowser.git ./manobrowser
  ```
  没有 git 时用 curl：
  ```bash
  curl -L https://github.com/ClawCap/ManoBrowser/archive/refs/heads/main.zip -o /tmp/manobrowser.zip
  unzip /tmp/manobrowser.zip -d /tmp/ && mv /tmp/ManoBrowser-main ./manobrowser
  ```

### Step 0.2 检查连接配置

确认环境中是否已配置 ManoBrowser 连接（任一命中即可）：
- `.mcp.json` 中有包含 `chrome_navigate` 相关的 MCP server 配置
- `config/mcporter.json` 中有对应配置
- 当前可用工具列表中已有 `chrome_navigate`（带任意前缀）

- **已配置** → 进入 Step 0.3
- **未配置** → 读取 `manobrowser/SKILL.md` 的「前置条件」章节，引导用户安装 Chrome 插件并完成配置

### Step 0.3 验证设备在线

调用 `chrome_navigate` 访问 `about:blank` 验证连接：
- ✅ **正常响应** → 前置完成，进入 Step 1
- ❌ **device not found** → 引导安装 Chrome 插件（参考 `manobrowser/SKILL.md`）
- ⚠️ **offline** → 提示打开 Chrome 并确认插件已启用

本文档中的工具名使用短名称（如 `chrome_navigate`）。实际调用时需加 MCP 实例前缀：
```
mcp__{实例名}__{工具短名}
```

## 2. 逐平台采集

### 照妖镜采集原则

照妖镜需要同时采集两个维度的数据来寻找反差：

| 维度 | 含义 | 来源 |
|------|------|------|
| **A面：人设层**（公开展示） | 用户主动发布的内容 | 帖子、投稿、动态、标记 |
| **B面：真相层**（私密偏好） | 用户被动暴露的偏好 | 收藏、点赞、喜欢列表 |

**A面和B面的差异 = 反差 = 照妖镜的核心数据。**

每个平台采集量上限（够找反差就行，不需要全量）：

| 数据类型 | 上限 |
|---------|------|
| 发布内容（A面） | 最新 50 条 |
| 收藏/点赞（B面） | 最新 50 条 |
| 关注列表 | 最新 100 个 |

### 通用采集流程

每个平台都遵循相同的基本流程：

```
1. chrome_navigate → 打开平台主页（登录态自动生效）
2. 定位「我的」/个人主页入口
3. 采集 A面：发布的内容（标题/摘要/标签）
4. 采集 B面：收藏/点赞/喜欢的内容（标题/摘要/标签）
5. 采集关注列表（关注了什么类型的账号）
6. 记录基础数据（粉丝数、关注数、作品数等）
7. chrome_close_tabs → 关闭采集页面
```

**采集方式**：使用 `chrome_execute_script` 在页面中运行 JS 脚本提取数据。需要滚动加载的用 `chrome_scroll` 分批加载。

> ⚠️ **脚本必须完整复制执行**——不要"简化"或"优化"JS脚本，龙虾容易丢失关键字段导致数据不全。

### 📕 小红书采集

**入口**：`https://www.xiaohongshu.com`

**A面 — 发布的笔记**：
1. 导航到用户主页（从首页点击头像/「我的」进入）
2. 在「笔记」tab 下，用 `chrome_execute_script` 提取笔记列表：
   ```javascript
   // 提取笔记卡片的标题和封面文字
   Array.from(document.querySelectorAll('.note-item, [class*="note"]')).slice(0, 50).map(el => ({
     title: (el.querySelector('.title, [class*="title"]') || {}).textContent?.trim() || '',
     desc: (el.querySelector('.desc, [class*="desc"]') || {}).textContent?.trim() || '',
     likes: (el.querySelector('[class*="like"], .like-count') || {}).textContent?.trim() || ''
   })).filter(n => n.title)
   ```
3. 如果不足 50 条，`chrome_scroll` 下滑加载更多，重复提取

**B面 — 收藏的笔记**：
1. 切换到「收藏」tab（点击收藏标签或导航到收藏页）
2. 用类似脚本提取收藏笔记的标题/描述
3. 滚动加载至 50 条

**关注列表**：
1. 点击「关注」数字进入关注列表页
2. 提取关注账号的名称和简介
3. 滚动加载至 100 个

> 💡 小红书的「发帖 vs 收藏」是照妖镜最佳数据源——发的是人设，收的是真相。

### 🎵 抖音采集

**入口**：`https://www.douyin.com`

**A面 — 发布的视频**：
1. 导航到个人主页（从首页点击头像进入）
2. 在「作品」tab，提取视频标题和描述
3. 用 `chrome_execute_script` 提取视频卡片信息

**B面 — 喜欢的视频**：
1. 切换到「喜欢」tab
2. 提取喜欢列表中的视频标题/描述
3. **注意**：抖音的「喜欢」默认公开，是反差挖掘的重要来源

**关注列表**：
1. 点击「关注」数字进入关注列表
2. 提取关注账号名称和简介

> 💡 抖音的「作品 vs 喜欢」反差最直观——发的是健身视频，喜欢的是深夜美食。

### 🐦 微博采集

**入口**：`https://weibo.com`

**A面 — 原创微博**：
1. 导航到个人主页
2. 筛选「原创」tab（过滤转发）
3. 提取微博文本内容，最新 50 条

**B面 — 收藏的微博**：
1. 导航到 `https://weibo.com/fav`（收藏页）
2. 提取收藏微博的内容摘要
3. **注意**：微博收藏是私密的，只有自己能看到——这是最"真实"的数据

**关注列表**：
1. 进入关注页面
2. 提取关注账号的名称和认证信息

> 💡 微博的「原创 vs 收藏」反差价值极高——原创是公开人设，收藏是私密真相。

### 📖 豆瓣采集

**入口**：`https://www.douban.com`

**A面 — 标记"想读/想看/想听"**：
1. 导航到用户主页
2. 进入「想读」「想看」「想听」列表
3. 统计数量 + 提取最新条目

**B面 — 标记"已读/已看/已听"**：
1. 进入「已读」「已看」「已听」列表
2. 统计数量 + 提取最新条目
3. **计算关键指标**：想读/已读的比值（这是豆瓣最出名的反差）

**评分分布**：
1. 提取用户的评分记录
2. 统计评分分布（是宽容型还是毒舌型）

> 💡 豆瓣的核心反差 = "想读 N 本 vs 已读 M 本"，简单但冲击力巨大。

### 📺 B站采集

**入口**：`https://www.bilibili.com`

**A面 — 投稿视频/动态**：
1. 导航到个人空间
2. 提取投稿视频列表（如有）
3. 提取动态内容

**B面 — 收藏夹**：
1. 进入「收藏」页面
2. 列出收藏夹名称（收藏夹名本身就很能说明问题）
3. 提取默认收藏夹中的视频标题

**关注列表**：
1. 进入关注列表
2. 提取关注的UP主名称和分区

> 💡 B站收藏夹的分类名是宝藏数据——"学习资料""以后再看""深夜食堂"比内容本身更能暴露真相。

### 采集注意事项

1. **先 A面后 B面**：先采公开内容，再采私密偏好，保持对比顺序
2. **记录数量**：每个维度的总数很重要（"想读89本"比"想读了一些书"有冲击力）
3. **提取标签/分类**：标签比正文更容易做聚类分析
4. **容错处理**：某个 tab 打不开或没内容就跳过，不要中断整个流程
5. **每个平台采集完保存**：写入 `mirror-reports/{日期}_raw_data.json`，防止中断丢失

## 3. 反差挖掘（⚠️ 核心步骤）

**这是照妖镜区别于普通画像工具的关键。** 不是总结"你喜欢什么"，而是找"你说一套做一套"。

### 分析框架

对每个用户，从以下角度寻找反差：

**A. 公开 vs 私密反差**
```
发帖说"今天又去健身房了💪" (3条)
收藏了"不运动也能瘦的10个方法" (7条)
→ 🪞 健身房去了3次，但收藏了7种不去健身房的理由
```

**B. 跨平台人设反差**
```
微博：转发社会新闻，评论时事，知识分子人设
小红书：全是穿搭、探店、美甲
→ 🪞 微博的你在思考人类命运，小红书的你在思考今天涂什么色号
```

**C. 理想 vs 现实反差**
```
豆瓣：标记了89本书"想读"
豆瓣：今年标记"已读"2本
→ 🪞 你的"想读"列表比"已读"长了44.5倍。按这个速度，读完要到2071年
```

**D. 表面兴趣 vs 真实时间分配**
```
关注了50个知识类博主，15个搞笑类博主
但点赞/互动90%在搞笑类
→ 🪞 你关注了50个"提升自我"的博主，但你的拇指只为段子停留
```

**E. 隐藏规律发现**
```
收藏的帖子中，67%包含"猫"
但从没发过关于猫的内容
→ 🪞 你嘴上没提过猫，但收藏出卖了你——一个深柜养猫人
```

### 挖掘规则

1. **必须基于数据**：每个反差结论都要有具体数字/内容支撑
2. **毒舌但不冒犯**：可以说"你嘴上和手上完全是两个人"，不可以人身攻击
3. **好笑 > 准确**：90%准但不好笑，不如70%准但很好笑
4. **具体 > 笼统**："收藏了7种不运动的方法"比"不爱运动"有趣100倍
5. **要有数字**：数字让反差更有冲击力（"44.5倍""67%""89本vs2本"）
6. **找3-7个反差**：太少没意思，太多疲劳。精选最戏剧化的

### 注意边界

**可以照的**：
- ✅ 兴趣反差、行为矛盾、人设反差、消费偏好、审美暴露

**不能照的**：
- ❌ 感情状态/性取向（除非用户自己公开）
- ❌ 经济状况评判、外貌评价、心理健康推测

**原则**：照妖镜让人「笑着社死」，不是「哭着被揭穿」。

## 写作风格指南

**好的**：
- "你关注了50个健身博主，但你的拇指只为奶茶停留"
- "豆瓣想读89本，已读2本。按这个速度你要读到2071年"
- "你的微博像《人民日报》读者，小红书像《VOGUE》读者，B站收藏像不能说"

**坏的**：
- "你的阅读量不够多"（太正经）
- "你可能存在认知失调"（心理分析）
- "你的消费观有问题"（道德评判）

**原则**：
1. 用对比制造笑点
2. 用数字制造冲击
3. 用具体制造共鸣
4. 结尾永远温暖
5. 不做道德评判——我们是照妖镜，不是道德法庭
